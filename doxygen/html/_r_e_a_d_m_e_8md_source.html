45 Installation
46 ============
47 **NOTE: If you can't install dependencies onto the computer you're using, create and copy the folders as mentioned below and then skip to [running the binary](#running-the-binary).**
48 
49 - Clone git repository into a folder called ```seprcph```
50 - Navigate to the local repository using the command line
51 - Depending on your Python installation run one of the following commands:
52  - ```python2.7 setup.py install```
53  - ```python setup.py install```
54 - Create a new folder in the relevant directory:
55  - Windows: C:/Users/Username/seprcph
56  - Linux ~/.config/seprcph
57 - Copy the ```assets``` and ```data``` folders from the local repository to the folder that you just created
58 - If you're on Linux, you can simply run ```seprcph``` from terminal. If you're on Windows, you can either install a better OS or do the following:
59  - Launch the game by running the command (using the python guidelines above):
60  ```python2.7 {path from current directory to the seprcph folder in the local repo}/main```
61 Running The Binary
62 ===========
63 If you are lucky enough to have the binary files, navigate to the folder that they are located in. Running the binary is as simple as double clicking it, alternatively, you can run it from terminal. This binary doesn't require any dependencies to be installed, so it will work on the university computers.
64 
65 Make sure that the binary remains in the folder it was provided in - all of the libraries required for running th binary are in that folder.

68 Building
69 ===========
70 It is possible to build the game into a binary, so that it can be run on other computers that don't have Pygame installed (such as the university PCs). In order to do this, you will need to have [cx_Freeze](http://cx-freeze.sourceforge.net/) installed.
71 
72 There is a script that will build the binaries for you - ```cx_build.py```.
73 
74 **NOTE: If you want to build a binary for Windows - you must do so on a Windows computer, the same is true for Linux.**
75 
76 Generating a binary is as simple as running:
77 
78 ```
79 python2.7 cx_build.py build
80 ```
81 
82 This will create a new ```build``` folder. This folder will contain a generated binary, as well as all the libraries required for running the game.

84 Generating Documentation
85 ==========
86 The code makes heavy use of [Doxygen](http://www.stack.nl/~dimitri/doxygen/) to document everything. We have a custom ```Doxyfile``` that is optimised for Python and our game. It will create a HTML website based upon the comments from within the code.
87 
88 We use an extension for Doxygen called [doxypypy](https://github.com/Feneric/doxypypy). In order to use this, we have set the ```INPUT_FILTER``` variable to use a script we have created.
89 
90 In order to generate the documentation, you need to be in the top level of the project (you should see the file ```Doxyfile``` in that directory) and run the command:
91 
92 ```
93 doxygen
94 ```
95 
96 We have been using Github Pages to host our documentation - you can read about them [here](https://pages.github.com/). The branch ```gh-pages``` in this repository contains the code for the website and the generated documentation.

98 Unit Tests
99 ==========
100 We have over 50 unit tests for our game. ```pygame``` is required to run them.
101 
102 Change into the top level directory of the project (the one containing the file ```setup.py```.
103 
104 ```
105 cd /path/to/seprcph
106 ```
107 
108 Then run:
109 
110 ```
111 python2.7 setup.py test
112 ```
113 
114 This will output the results of all of the tests. Make sure you rerun the tests after each change you make.

116 Code Checking
117 ==========
118 We have tried hard to stick to the [PEP8](https://www.python.org/dev/peps/pep-0008/) standard and have tests written to help with this. In order to run the code checking tests, you will need to have [PEP8](https://pypi.python.org/pypi/pep8) and [pylint](http://www.pylint.org/).
119 
120 A script has been written to help with the code checking - but it will only work on *NIX systems (or Windows if you have Cygwin + Bash). You can run it as follows:
121 
122 ```
123 scripts/check seprcph
124 ```
125 
126 Take a look inside of the script to see what commands are being run - in case you want to run them on your own.
